Tema 1: Elementos de Páginas Web

1.origenes de la web

La Web 2.0, emergiendo a principios de los años 2000, representa una evolución de la web centrada en la interacción y la colaboración. A diferencia de la Web 1.0, que era principalmente estática y se centraba en la publicación de contenido, la Web 2.0 permite a los usuarios crear, compartir y modificar contenido. Plataformas como redes sociales (Facebook, Twitter), blogs y wikis son ejemplos clave. La democratización de la creación de contenido y el surgimiento de comunidades en línea son características distintivas de esta etapa.

2. Web 3.0 3

La Web 3.0, también conocida como la web semántica, busca hacer que los datos en la web sean más comprensibles y utilizables por máquinas. Se centra en el uso de tecnologías como la inteligencia artificial, el aprendizaje automático y blockchain. La Web 3.0 pretende proporcionar experiencias más personalizadas y conectar información de manera más efectiva, permitiendo un entorno donde los usuarios puedan interactuar con los datos de forma más intuitiva y segura

3. La W3C y los estándares

La World Wide Web Consortium (W3C) es una organización que se encarga de desarrollar estándares y directrices para asegurar el crecimiento y la interoperabilidad de la web. Fundada por Tim Berners-Lee, el inventor de la web, la W3C trabaja en la creación de protocolos y especificaciones que permiten que los navegadores y otros servicios web funcionen de manera coherente. Sus estándares son fundamentales para mantener la accesibilidad, la usabilidad y la seguridad en la web.

II. Lenguajes de etiquetas

1. SGML SGML (Standard Generalized Markup LanguageSGML es un estándar para definir lenguajes de marcado. Fue creado en la década de 1980 y se utiliza para describir la estructura de documentos. SGML es flexible y permite a los usuarios definir sus propias etiquetas. Aunque no se utiliza directamente en la web, sentó las bases para lenguajes posteriores, como HTML y XML.

2. XML

XML (eXtensible Markup Language)XML es un lenguaje de marcado que permite definir y estructurar datos de manera que sea tanto legible para humanos como para máquinas. A diferencia de HTML, que está diseñado para mostrar datos, XML se centra en el transporte y almacenamiento de datos. Es ampliamente utilizado para intercambiar información entre sistemas y aplicaciones

3. HTML

HTML (HyperText Markup Language)HTML es el lenguaje de marcado estándar para crear páginas web. Desarrollado en la década de 1990, HTML permite estructurar contenido mediante el uso de etiquetas. Con HTML, se pueden definir encabezados, párrafos, enlaces, imágenes y otros elementos en una página web. HTML ha sido fundamental en el desarrollo de la web tal como la conocemos hoy.

4. XHTML

XHTML (eXtensible HyperText Markup Language)XHTML es una versión más estricta y basada en XML de HTML. Introducido a finales de los años 90, combina la flexibilidad de HTML con la estructura rigurosa de XML. XHTML requiere que todos los documentos estén bien formados, lo que mejora la interoperabilidad entre diferentes navegadores y dispositivos. Aunque se utiliza menos hoy en día, estableció estándares importantes para la codificación de páginas web.

5. HTML5

HTML5 es la última versión del lenguaje HTML, lanzada oficialmente en 2014. Introduce nuevas características y elementos que permiten mejorar la multimedia y la interacción en la web, como audio, video y gráficos. HTML5 también simplifica el uso de formularios y mejora la accesibilidad. Su diseño es más compatible con dispositivos móviles y ofrece soporte para aplicaciones web más complejas

III. Otros lenguajes relacionados al desarrollo web

h<2>1. CSS/CSS3

CSS/CSS3CSS (Cascading Style Sheets) es un lenguaje utilizado para describir la presentación de documentos HTML. Permite separar el contenido de la estructura visual, lo que facilita el diseño y la adaptabilidad de las páginas web. CSS3 es la última versión y añade nuevas características como animaciones, transiciones y un modelo de diseño más flexible, permitiendo un mayor control sobre la apariencia y la disposición de los elementos.

2. JavaScript

JavaScript es un lenguaje de programación de alto nivel, fundamental para la interacción en la web. Permite a los desarrolladores crear dinámicas en las páginas, como menús interactivos, validación de formularios y contenido actualizado sin necesidad de recargar la página. JavaScript se ha convertido en un pilar del desarrollo web, especialmente con el auge de frameworks y bibliotecas como React, Angular y Vue.js.

3. Ajax

Ajax (Asynchronous JavaScript and XML) es una técnica que permite la actualización asíncrona de contenido en una página web. Facilita la comunicación entre el navegador y el servidor sin recargar la página completa, mejorando la experiencia del usuario. Ajax combina HTML, CSS, JavaScript y, a menudo, JSON (en lugar de XML) para intercambiar datos de manera eficiente y crear aplicaciones web más interactivas .

5. Navegadores

Los navegadores web son programas que permiten a los usuarios acceder y visualizar contenido en la web. Ejemplos comunes incluyen Google Chrome, Mozilla Firefox, Safari y Microsoft Edge. Cada navegador puede interpretar HTML, CSS y JavaScript de manera ligeramente diferente, lo que puede afectar la apariencia y funcionalidad de las páginas web. Los desarrolladores deben asegurarse de que sus aplicaciones funcionen correctamente en múltiples navegadores.

6. Motores renderizados

Los motores de renderizado son componentes de los navegadores que interpretan el código HTML, CSS y JavaScript para mostrar las páginas web. Cada navegador utiliza su propio motor de renderizado. Algunos ejemplos son: Blink: Utilizado por Google Chrome y Microsoft Edge. Gecko: Usado por Mozilla Firefox. WebKit: Utilizado por Safari. Estos motores determinan cómo se visualiza y se comporta una página web, lo que puede influir en el rendimiento y la compatibilidad de las aplicaciones.

IV. Herramientas de desarrollo

1. Softwa WYSIWYG (What You See Is What You Get) se refiere a programas que permiten a los desarrolladores y diseñadores crear contenido web sin necesidad de escribir código directamente. Estos editores muestran el diseño final en una interfaz visual, lo que facilita la creación de páginas web. Algunos ejemplos populares incluyen: Otras alternativas

Adobe Dreamweaver: Ofrece una interfaz gráfica junto con la posibilidad de editar código.

Microsoft Expression Web: Permite crear y editar sitios web de manera visual.

WordPress: Aunque es un sistema de gestión de contenido (CMS), muchas plantillas y editores visuales permiten un enfoque WYSIWYG. Estas herramientas son ideales para principiantes o para quienes prefieren un enfoque más visual en el diseño web.

i. Figma

Figma es una herramienta de diseño colaborativo basada en la nube que permite a los diseñadores crear interfaces de usuario (UI) y prototipos interactivos. Algunas de sus características incluyen: • Colaboración en tiempo real: Varios usuarios pueden trabajar en el mismo diseño simultáneamente. • Prototipado: Permite crear prototipos interactivos para simular la experiencia del usuario. • Componentes reutilizables: Facilita la creación y mantenimiento de elementos de diseño consistentes. Figma es popular entre equipos de diseño y desarrollo debido a su enfoque colaborativo.

ii. InVision

InVision es otra herramienta de diseño y prototipado que permite a los equipos crear prototipos interactivos y colaborar en el proceso de diseño. Sus características incluyen: • Prototipos interactivos: Facilita la creación de prototipos que simulan la funcionalidad de una aplicación. • Feedback y comentarios: Permite a los equipos recibir comentarios directamente en los diseños. • Integraciones: Se puede integrar con otras herramientas como Sketch y Photoshop. InVision es ideal para diseñadores que buscan un flujo de trabajo ágil y colaboración efectiva.

iii. Zeplin

Zeplin es una herramienta de colaboración entre diseñadores y desarrolladores que facilita la transición de diseños a código. Sus principales características son: • Generación de especificaciones: Genera automáticamente estilos, medidas y recursos a partir de los diseños. • Integraciones: Se integra con herramientas de diseño como Figma, Sketch y Adobe XD. • Colaboración: Permite a los desarrolladores acceder a todos los detalles necesarios para implementar los diseños. Zeplin ayuda a cerrar la brecha entre el diseño y el desarrollo, facilitando la comunicación.

iv. GitHub

GitHub es una plataforma de desarrollo colaborativo que utiliza Git para el control de versiones. Es ampliamente utilizada en el desarrollo de software y ofrece: • Control de versiones: Permite a los desarrolladores rastrear cambios en su código y colaborar con otros. • Repositorios: Los proyectos se almacenan en repositorios, que pueden ser públicos o privados. • Integración continua: Se pueden configurar flujos de trabajo de integración continua (CI/CD) para automatizar pruebas y despliegues. • Colaboración: Facilita la colaboración a través de "pull requests", donde los cambios propuestos pueden ser revisados y discutidos. GitHub es esencial para la gestión de proyectos de desarrollo y es ampliamente utilizado en la comunidad de software.
